05B / Expand Detail Table

傳統 table：點 row 展開 detail

預設模式：點 row 展開 detail；右滑只開 action。
專案環境位置狀態版本最後報到
點 row：展開 detail 並自動打開 action rail。右滑 row：只打開 action rail，不展開。點另一 row 時會補償新箭頭位置。

12 / Header

共通頁首 / Clean Shell Header

Header 是乾淨的共通導航殼，不是 runtime dashboard。它不顯示 PROD/DEV、Script、CSS、version；左側 brand 本身就是回首頁，右側只放登入/登出這類帳號動作。 外部 App 需要時可保留「回 SKHPS 首頁」。

參考目前 header.js：實際掛載點是 <header id="header">，左側 SKHPS / Plastic Surgery，右側預設只有登入。

13 / Footer

Runtime Status Footer

Footer 是低調 runtime 狀態列。左側保留環境摘要，中間保留不搶眼版本號；右側維持目前 footer.js 的原始 traffic light 樣式，不改成 Theme40 badge 或大膠囊。

14 / Rules

使用規則

①
規則 1
頁面不得新增自己的 button/card/table/timer 樣式。
②
規則 2
RWD 只分三層：Desktop、Tablet、Mobile。
③
規則 3
頁面特殊 CSS 只能做少量 override。
④
規則 4
手機拖曳不要只用 dragstart/drop，要用 Pointer Events。
⑤
規則 5
統一用途變色：藍主色、綠成功、黃警告、紅危險。

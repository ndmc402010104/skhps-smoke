00 / Overview

UI 設計不是裝飾，是工作流程的空間安排

這份 smoke 是給 SKHPS 用的 UI 設計閱讀館。它不是要你背設計名詞，而是讓你慢慢建立判斷：什麼畫面該寬、什麼該窄；什麼該放在 header、footer；table 到底什麼時候該用、什麼時候會害人；按鈕、表單、卡片、色彩、手機版要怎麼想。

核心觀念：UI 不是把畫面做漂亮而已。UI 是把使用者的工作流程排成一個合理的空間，讓他不用想太多就知道下一步在哪裡。

為什麼你之前會覺得畫面只有中間 1/3？

因為很多模板是從「文章 / Landing Page / 表單頁」的世界來的。那個世界強調閱讀舒適，所以會把文字收在中間，不讓一行太長。可是 SKHPS 很多頁面不是文章，而是醫院內部工具、後台、清單、表格、管理頁。這類頁面如果還套文章頁容器，就會變成：左右空白很多，真正的工作區很小。

錯誤直覺

所有頁面都套同一個 max-width，覺得這樣比較統一。

結果：表單還可以，後台清單直接變擠。

正確直覺

先判斷頁型，再決定寬度、資訊密度、元件形式。

結果：閱讀頁好讀，後台頁好操作，報表頁有足夠空間。

這份 smoke 的閱讀方式

先看大方向：版面配置、header、footer，先決定頁面的骨架。
再看資料呈現：table、card、detail、資訊密度。
最後看操作元件：button、form、feedback、mobile、motion。
這份 smoke 的定位

不是正式元件庫。正式元件庫是 Theme40 / CSS Sheet。

這是設計思考教材。它讓你在改 SKHPS 前先知道：為什麼要這樣改、什麼時候不該這樣改。

04 / Table

Table 應用：表格不是 Excel，管理頁也不是資料庫 dump

這一章是你最在意的。橫向捲動不是錯，但它不該是一般管理頁的主要閱讀方式。大多數 SKHPS 頁面應該採用「主資訊表格 + detail 展開」，而不是把所有欄位攤成 Excel。

一、先分清楚三種 table

Management List

管理清單

日常操作用。目標是找資料、看狀態、做動作。應避免橫拉。

Detail Table

主列 + detail

主列放 A 層資訊，detail 放 B 層資訊。最適合 SKHPS 多數頁面。

Report Table

報表表格

比對、稽核、匯出用。可以更寬，必要時才允許橫拉。

四、欄位分層規則

A 層：主表一定要看到 名稱、狀態、位置、關鍵數值、最後更新、動作。
B 層：detail 才看 版本、來源、備註、次要 ID、建立者、修改者。
C 層：編輯頁 / modal 才看 長備註、完整歷程、權限細節、稽核 log。
SKHPS table 規範：一般管理頁禁止把橫向捲動當成主要閱讀方式。先拆主次，再決定 detail。只有報表 / 稽核 / 匯出預覽才允許橫拉 fallback。

手機操作：不是把桌機縮小，而是重排任務

手機版最常見的錯誤，是把桌機 table 硬縮小。結果就是字小、欄位擠、按鈕難點。手機應該是重新編排任務，不是縮圖。

一、手機只保留 A 層資訊

桌機可以有 5–6 個視覺欄位，手機通常只能承受 2–3 個主資訊。其他資訊應該進展開層。

二、手機操作要避免誤觸

手機上危險按鈕不能太靠近常用按鈕。刪除、停用、清空應該需要 confirm，或放在展開層 / 編輯頁深處。

三、Swipe action 要清楚

Swipe 很好用，但不是每個使用者都知道可以滑。所以最好同時提供可見的操作入口，例如「更多」或三點選單。你之前做的 swipe table 已經有方向，但仍要避免到處都出現怪 cursor 或拖曳觸發。

手機主畫面：只看主角、狀態、關鍵值。
展開層：放次要資訊與次要操作。
危險操作：不要跟取消 / 儲存太近。
拖曳排序：只能在 handle 上觸發，不要整列都變拖曳。

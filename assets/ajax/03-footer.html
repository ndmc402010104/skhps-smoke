03 / Footer

Footer：低調的 runtime 狀態與信任感

Footer 在 SKHPS 很重要，因為你不是一般展示網站。你的系統有 local / dev / prod，有 Apps Script / Cloudflare / Supabase，有 loading gate，有 CSS runtime。這些資訊需要存在，但不能喧賓奪主。

一、Footer 的定位

Footer 是「低調可信的狀態列」。它不是主要操作區，也不是主視覺。它是讓你在出問題時可以快速判斷：現在跑在哪個環境？資料來源是哪裡？Gate / Backend / CSS 是否正常？

Footer 示範：低調 runtime
PAGE DEV ｜ Runtime DEV ｜ Script LOCAL
v2.2.18-dev
🟢 Gate🟢 Config🟢 Backend🟢 CSS🟢 Data

二、Footer 為什麼不要做太漂亮？

因為 footer 的工作是「穩定存在」，不是吸引注意。太大的 badge、太亮的色彩、太多卡片，都會讓頁尾變成第二個 dashboard。這會干擾主要工作畫面。

SKHPS 很適合 traffic light footer。因為紅黃綠狀態燈能快速提醒問題，但平常不會干擾閱讀。

三、Footer 要顯示什麼？

環境：PAGE / Runtime / Script 是 local、dev 還是 prod。
版本：中間低調顯示即可，不要搶視線。
狀態燈：Gate、Config、Backend、CSS、Data。
展開 panel：詳細 log 可以藏在 panel，不要全部攤開。

四、Footer 的切換問題

你之前遇到 local / dev / prod 切換錯誤，本質不是 footer 視覺問題，而是 footer 同時扮演了「狀態顯示」與「環境切換導航」。這種功能可以保留，但必須清楚：

local 按下去 → dev
dev 按下去 → prod
prod 按下去 → local 或回對應環境

而且不能 fallback 到 index 錯頁。
Footer link 必須知道目前 pageId / appId / route。
設計提醒：Footer 可以有環境切換，但點擊目標要小心。狀態燈是「看」，環境切換是「做」，兩者要視覺上分清楚。

版面配置：先分頁型，再談美感

布局最容易犯的錯，是先問「好不好看」，但真正該先問的是：「這頁是拿來做什麼的？」不同任務需要不同寬度、不同資訊密度、不同視線路徑。

一、三種桌機頁型

Reading / Form

窄頁

適合說明、設定、單一編輯、表單。目標是好讀、好填。寬度約 820–1000px。

Admin List

寬頁

適合後台、清單、CSS Setting、人員名單、專案管理。目標是掃描與操作。寬度約 1280–1600px。

Report / Audit

滿版頁

適合稽核、報表、比對、匯出預覽。目標是容納資料，但仍要分層，不是無腦塞欄位。

二、同一套視覺，不等於同一個寬度

「統一設計」不是每個頁面都一樣寬，而是背景、字體、色彩、按鈕、卡片語言一致。寬度應該依任務調整。

體感：三種容器
窄頁 / Reading：適合填寫、閱讀、單一設定。這裡如果太寬，眼睛會疲勞。
寬頁 / Admin：適合管理清單。這裡如果太窄，就會浪費螢幕，表格也會擠。
滿版 / Report：適合報表與比對。但它應該是少數頁面，不是日常管理頁的預設。

三、SKHPS 的頁型建議

首頁：中寬，不要滿版。首頁是導航，不是資料表。
CSS Setting：寬頁。它是管理工具，不能只用中間一條。
quick-login 人員後台：寬頁。主表只放關鍵欄位，其他進 detail。
dressing inventory：寬頁 + detail。主表看庫存狀態，批號進 detail。
QR sign 紀錄：看用途。日常查詢用寬頁；匯出稽核用報表頁。
一句話：不要再用單一 .sk-page 解決所有頁面。SKHPS 應該有 reading、admin、report 三種頁型。

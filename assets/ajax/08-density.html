08 / Information Density

資訊密度：一眼望穿不是資訊少，是資訊分層

你說的「一眼望穿」非常重要。它不是把資料變少，而是讓第一眼只承受該承受的資訊。其他資訊不是消失，而是進到第二層、第三層。

一、三層資訊模型

A Layer

第一眼資訊

使用者掃每一列都需要看到。名稱、狀態、位置、關鍵值、最後更新。

B Layer

展開資訊

使用者有興趣才看。備註、來源、版本、次要 ID、建立者。

C Layer

深層資訊

編輯或稽核才看。完整歷程、長備註、權限細節、log。

二、密度不是越低越好

有些設計文章會推崇大量留白，但後台系統如果留白過頭，會變成一頁只能看到 3 筆資料，效率很差。SKHPS 需要的是「舒適但有效率」的密度。

醫院內部工具的密度：不能像行銷網站那麼鬆，也不能像 Excel 那麼滿。要讓使用者在 5–10 秒內找到目標。

三、如何提高密度但不變亂？

固定視線錨點：每列最左邊永遠是主角。
次要資訊用小字：放在主標下面，不要另開一欄。
狀態用 badge：但 badge 顏色不要太多。
動作固定在右側：使用者不用找按鈕。
detail 統一位置：不要每種頁面展開方式都不同。

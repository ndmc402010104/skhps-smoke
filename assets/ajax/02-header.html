02 / Header

Header：頁首不是 runtime dashboard，是方向感

Header 的工作不是把所有資訊都塞上去。Header 的工作是讓使用者知道：我在哪個系統？我現在在哪一頁？我可以去哪裡？我是誰或登入狀態如何？

一、Header 的四個任務

身份識別：SKHPS / Plastic Surgery / 外部 App 名稱。
方向感：回首頁、目前頁面、麵包屑或簡短標題。
帳號動作：登入、登出、使用者身份。
少量全域動作：例如回 SKHPS 首頁、進後台，但不能塞太多。

二、Header 不應該做什麼

不要把 Header 變成飛機駕駛艙。環境、版本、CSS、Backend、Gate、Data source、debug 狀態，這些應該低調放 footer 或 runtime panel，不該占用 header。

三、外部 App 的 Header 要更克制

外部 app 的 header 不需要把水庫 runtime 的資訊重複一次。它只要告訴使用者目前是誰、屬於 SKHPS、可以回到 SKHPS 首頁即可。

錯誤

Header 放滿環境燈、版本、CSS 狀態、Backend 狀態、資料來源、debug 開關。

結果：頁面一開始就很吵，真正的頁面標題反而不清楚。

正確

Header 保持乾淨：品牌、頁面名稱、主要導航、登入狀態。

Runtime 狀態往 footer / panel 下沉。

四、SKHPS Header 規範草案

Header 必放：
- 左側：SKHPS brand / app name
- 中間或副標：目前頁面名稱
- 右側：登入 / 登出 / 回首頁

Header 禁止：
- 大量 runtime 狀態
- 大量版本資訊
- 多層複雜選單
- 每個外部專案自行設計一套 header
